                              KEY COMPONENTS
================================================================================

1. HARMONY ENCODING
   • OpenAI's unified format for multimodal + tool + reasoning
   • Handles stop tokens, tool invocation, role switching
   • More efficient than manual prompt engineering

2. vLLM SERVER
   • Single H100 GPU deployment
   • Tensor parallelism: 1 (single GPU)
   • KV cache quantization: FP8
   • Max context: 65536 tokens
   • GPU memory util: 96%

3. PERSISTENT JUPYTER KERNELS (16 parallel)
   • Long-lived Python sandbox for each worker thread
   • State persists across attempts
   • Pre-loaded with: math, numpy, sympy, mpmath
   • ~500ms per execution (vs 2s with subprocess)

4. STREAMING COMPLETION
   • Token-level streaming from vLLM
   • Collect logprobs for confidence signal
   • Early stopping on answer pattern \boxed{...}
   • Timeout protection: 5-7s per problem

5. ENTROPY WEIGHTING
   • Token-level logprobs → distribution entropy
   • Low entropy = high model confidence
   • Weight = 1 / entropy (inverse confidence)
   • Simple, interpretable, no learned weights

6. EARLY STOPPING
   • Stop all attempts if 4+ agree on answer
   • Saves 15-20% compute with minimal score loss
   • Dynamic time budgeting across 50 problems
